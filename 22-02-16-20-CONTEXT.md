## 3. MODIFICATIONS LIÉES AU STATE ET AU ROUTAGE (App.jsx)

### Élévation d'état (`Lifting State`)
Pour permettre à n'importe quel panel d'engager la communication avec un agent spécifique, la `CommsConsole` a été modifiée. 
Le state `activeAgentId` qui était local à `CommsConsole` a été remonté dans `<App />`.

**Mécanique d'Uplink depuis `AgentsPanel` :**
1. Un clic sur le bouton "UPLINK" dans une carte agent (depuis `AgentsPanel.jsx`) déclenche le callback `onSelectAgent(id)`.
2. `<App />` intercepte et fait deux choses : `setActiveAgentId(id)` et `setActiveTab('comms')`.
3. Le routeur passe sur la vue `CommsConsole`, qui reçoit le `activeAgentId` forcé en prop et ouvre immédiatement le canal avec cet agent.

## 4. DÉCISIONS TECHNIQUES CONCERNANT LE DND-KIT (Drag & Drop)

L'implémentation du Kanban dans `TasksPanel.jsx` se devait d'être résiliente par rapport au flow asynchrone arbitraire généré par `useSimulation.js`. 

**Le Conflit Data State vs UI State :**
- `useSimulation` regénère perpétuellement son objet `tasks` Kanban.
- Utiliser directement cet objet dans `dnd-kit` brise le drag en pleine course si un `setInterval` tick.

**La Solution :**
1. `TasksPanel` gère un state LOCAL strict `localTasks` qu'il popule uniquement à l'initialisation (via la prop initiée par `simulation.tasks`).
2. Les interactions de déplacement (`handleDragStart`, `handleDragOver`, `handleDragEnd`) mutent uniquement ce state interne `localTasks` à l'aide de l'utilitaire `arrayMove`.
3. Cela garantit une physique fluide (`DragOverlay`) sans glitch de concurrence. 

**Intégration Backend (TODO Future) :**
Dans un environnement de production (ex: branché sur MissionDeck/OpenClaw), la fonction `handleDragEnd` devra émettre un événement WebSocket/API (ex: `PATCH /tasks/TSK-A2B4 {"column": "IN_PROGRESS"}`), et le cache Data (ex: `react-query`) répercutera l'état réel.

## 5. DÉCISIONS ESTHÉTIQUES ET SOVEREIGN GLASS

Les nouvelles interfaces poussent l'esthétique du projet plus loin sans violer les fondations.

- **Fonds Alternatifs** : Utilisation intensive de `bg-black/40` ou `bg-[#0A0A0F]` avec des bordures `border-white/5` pour créer une profondeur modulaire à l'intérieur des panels existants (ex: colonnes du Kanban).
- **Tooltips d'intensité** : La nouvelle `ThreatMatrix` dans `IntelPanel` utilise des popovers absolus (`opacity-0 group-hover:opacity-100`) pour afficher les métriques fines sans surcharger l'UI principale.
- **Scrollbars et Overflow** : Le scroll principal de la fenêtre est cassé volontairement (`overflow-hidden` sur le main `App.jsx`). Chaque panel possède son propre `flex-1 overflow-y-auto custom-scrollbar` permettant une navigation en tunnel très focalisée.
